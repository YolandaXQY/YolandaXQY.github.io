2. http 请求成功，angular订阅之后进error

场景: angular 在调用下载文件的接口, 接口返回的数据类型是·二进制流·, 一开始非常惆怅为什么接口调用成功但是却在success的回调函数中没有任何体现，我怕是请求问题，去看了请求，我打印出来,返回的值是Observable对象，是可订阅的，后来想肯定是进error了，果然打印出”失败”；网上去搜相关的，说是ajax请求成功，但是进error,是因为dataType没有设置；后来一想，因为angular中默认的返回类型就是json, 但是这次返回的是二进制流不匹配导致的，所以在请求参数中加了 {responseType: ‘blob’} 就成功了。

(一) 开发桌面应用程序框架选型

由于本人只会html,css,js,部门就我一个写web的，然后就想要寻找一种框架，能够使用html,css,javascript等技术快速搭建一个原生的应用。
经过搜寻，下面几种框架都是可以跨应用的。

  1. ioinc: 基于HTML5开发的移动端APP的原生应用开发框架。https://ionicframework.com/
  2. hex: 网易开发的一个桌面应用程序，其产品主要应用于有道词典。http://hex.youdao.com/zh-cn/tutorial/index.html (好像已经不再更新了)
  3. NW(node webkit): 与hex实现原理差不多，但是NW是在hex之前出现的，也就算鼻祖。都是利用HTML + javascript开发的桌面应用程序，主要是实现方式就是在应用的配置文件中配置入口。
  4. electron: https://electronjs.org/docs 这个应用的作者与NW是同一个人，electron是在NW之后，与NW明显不同的地方是，它的入口是一个脚本，通过手动创建窗口的方式，集成了node, 更符如今的开发方式。

综上所述，我们采用了electron的框架。

1. border-radius 圆角

https://www.zhangxinxu.com/wordpress/2015/11/css3-border-radius-tips/
https://blog.csdn.net/YaoDeBiAn/article/details/78295725

border-radius的圆角计算原理： 首先遵从两个原理，一是大值特性，二是等比例特性。

1) 大值特性: 如果border-radius设置的值很大的适合，只会采用能够渲染圆角的大小渲染。
比如说，width:300px;height:200px;设置成border-radius:250px;只会按照<200px去渲染，因为如果太大了就没法渲染了。
2) 等比例特性
border-radius: 300px/200px;需要遵从水平/垂直= 3:2进行渲染。

####### border-radius的写法:
1) border-radius: 200px 10px 0 30px/200px 10px 0 30px; 这是原始的最复杂的写法，即： 水平方向/垂直方向，依次是左上，右上，右下，左下；
2) border-top-left-radius: 300px 100px;
border-top-right-radius: 300px 100px;
border-bottom-right-radius: 300px 100px;
border-bottom-left-radius: 300px 100px;
3) border-radius: 300px;
4) border-radius: 50%;

2. box-shadow

1) box-shadow: h-shadow v-shadow blur spread inset/outset; 参数分别是: 水平距离/垂直距离/模糊距离/阴影尺寸/内部或者外部阴影

2) 实现四周阴影: box-shadow: 0 0 15px 4px #999; 需要将h-shadow,v-shadow 设置为 0

3) 只有底部阴影的: box-shadow: 0 7px 0 #999; 但是不能设模糊距离
只有顶部阴影: box-shadow: 0 10px #999 inset;
只有左部阴影: box-shadow: 10px 0 #999 inset;
只有右部阴影: box-shadow: 10px 0 0 #999;
