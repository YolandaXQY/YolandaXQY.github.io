获取Dom元素的高度

Veröffentlicht am 2019-01-31

//获取盒子客户端的高度
box.clientHeight //输出110 (内容高度+padding * 2)

clientHeight表示内容的实际高度,包括padding
offsetHeight表示元素的实际高度，包括内容的实际高度+边框的高度,可用尺子量的。

scrollHeight 为子元素的实际高度加margin，也就是子元素的offsetHeight + margin*2。如果该元素没有产生滚动条，即其子元素的高度少于该元素的高度，则scrollHeight等于该元素的clientHeight

//如果box的父容器没有设置定位，则box.offsetLeft是相对body而言，否则是根据离它最近且有设置定位的父容器而言
box.offsetLeft //获取box盒子相对它的父容器水平的偏移
box.offsetTop //获取box盒子相对它的父容器垂直的偏移

//当一个容器设置高度，如果该容器有滚动条，则该容器的内容高度要减去滚动条的高度,滚动条的高度一般为17px

Veröffentlicht am 2019-01-30 | Edited on 2019-01-31

1. 数据库long类型的，JS 解析精度损耗

JS对number类型的只能精确到17位，解决方案只能是后端转成string给前端

2. http 请求成功，angular订阅之后进error

场景: angular 在调用下载文件的接口, 接口返回的数据类型是·二进制流·, 一开始非常惆怅为什么接口调用成功但是却在success的回调函数中没有任何体现，我怕是请求问题，去看了请求，我打印出来,返回的值是Observable对象，是可订阅的，后来想肯定是进error了，果然打印出”失败”；网上去搜相关的，说是ajax请求成功，但是进error,是因为dataType没有设置；后来一想，因为angular中默认的返回类型就是json, 但是这次返回的是二进制流不匹配导致的，所以在请求参数中加了 {responseType: ‘blob’} 就成功了。

docker + nginx 部署

  1. Run npm run build-prod, 会打出一个dist的文件夹，压缩成dist.zip;

  2. 以服务器192.168.9.27为例，进行docker发布;

    • cd /进入根目录;
    • cd /home/ms;
    • mkdir cc-app;
    • 通过 rz 命令上传 dist.zip/Dockerfile/nginx.conf, 并 unzip dist.zip解压文件夹;

      Dockerfile/nginx.conf与package.json同一级目录。

    • 打镜像: docker build -t cc-app:test .

      注意： 后面cc-app:test 是指的是 容器名称:tag, 一个容器可以存在多个tag

    • 运行镜像: docker run -idt -p 8081:80 --name cc-app cc-app:test

      注意: 8081:80 前者是此项目访问是的端口，后者是nginx的端口。

  3. docker的服务停止和删除

    如果要重新部署，必须先停掉服务，再删除容器和镜像，重新执行第二步命令（存放的文件夹可以不删哦）

    • 停止docker容器: docker stop cc-app;
    • 删除docker容器: docker rm cc-app;
    • 删除镜像image: docker rmi -f cc-app:test;

  4. 其他命令

    • 查看所有容器: docker ps -a;
    • 查看所有image: docker images;
    • 复制镜像: docker tag cc-app-test:test cc-app:test;
    • 删除文件: docker -f dist.zip;
    • 删除文件夹: docker -rf dist

H5 缓存

Veröffentlicht am 2019-01-11 | Edited on 2019-01-22

1. cookie

存储大小: 4kb;
特点: 1. cookie数据始终在同源的http请求中携带，即使不需要，会在浏览器和服务器之间来回传递。

2. 通常存用户的加密信息。

2. sessionStorage

存储大小: 5M
特点: 会话级别的本地缓存，当页面关闭，缓存清除。（爬虫不能爬到）

3. localStorage

存储大小: 5M
特点： 本地存储，永久保存，需要手动删除。（爬虫不能爬到）

4. 如果溢出怎么办?

溢出会报异常，不推荐很多数据走缓存。
